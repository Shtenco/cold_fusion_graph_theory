## 2. Главная формула

<!-- F14 -->
$$
\mathfrak U
=
\left(
\mathcal H,\,
L,\,
\Theta,\,
\{f_i\}_{i=1}^{6},\,
e^{-\sigma n}
\right)
\Rightarrow
\text{конечная, единая, абсолютно сходящаяся физика}
$$

В этом модуле формула применяется к LENR-гипотезе: холодный синтез трактуется как графовое снижение эффективного барьера в коллективной среде, а не как инструкция по созданию устройства.

## 4. Связь с `info_graph_theory`

Модуль холодного синтеза не является отдельной физикой. Он является частным сектором общей теории информационного графа:

```text
info_graph_theory
└── cold_fusion_graph_theory
    ├── графовые параметры
    ├── барьерное подавление
    ├── числовой баланс
    └── CAS-проверка README
```

Общая теория объединяет четыре взаимодействия через спектр лапласиана `L`. LENR-модуль использует тот же каркас, но рассматривает только эффективный барьерный сектор.

## 5. Совместимость с ОТО, струнной логикой и четырьмя взаимодействиями

Модуль не заменяет Общую теорию относительности и не утверждает, что теория струн неверна. Он использует совместимую иерархию:

- на фундаментальном уровне есть информационный граф;
- на спектральном уровне есть лапласиан `L`;
- на геометрическом уровне возникает эффективная метрика;
- в континуальном пределе гравитационный сектор должен сводиться к ОТО;
- в модовом описании частицы являются возбуждениями, что структурно похоже на струнную идею спектра возбуждений;
- четыре взаимодействия являются разными секторами одного спектрального действия.

Единая секторная запись:

<!-- F15 -->
$$
S_{\mathrm{tot}}
=
S_{\mathrm{em}}
+S_{\mathrm{weak}}
+S_{\mathrm{strong}}
+S_{\mathrm{grav}}
$$

Графовая интерпретация:

| Сектор | Интерпретация |
|:--|:--|
| `S_em` | фазовые U(1)-моды |
| `S_weak` | смешивание спектральных подпространств |
| `S_strong` | внутренние стабилизаторы цветового типа |
| `S_grav` | геометрия лапласиана и теплового ядра |

## 6. Параметры графа

`Theta=(K,p,N)`, где `K=6`, `p=4.8027e-42`, `N=4.197668e121`.

Инфотонная нормировка:

<!-- F01 -->
$$
\chi_I=2K^2+2K+1,\qquad \chi_I(6)=85
$$

Структурное тождество:

<!-- F02 -->
$$
f_2f_5=\ln K\cdot\frac{K}{\ln K}=K
$$

Критический режим:

<!-- F09 -->
$$
\frac{Kp}{Kp}=1
$$

Численный аудит дополнительно проверяет, что `Kp` и `N^(-1/3)` имеют относительное расхождение меньше `0.002`.

## 8. Абсолютная сходимость

Проверяемый ряд:

<!-- F10 -->
$$
\lim_{n\to\infty}
\frac{(n+2)^q e^{-\sigma(n+1)}}{(n+1)^q e^{-\sigma n}}
=e^{-\sigma}<1
$$

При `sigma>0` экспоненциальный регулятор подавляет любой полиномиальный рост. Это связывает LENR-модуль с общей формулой `mathfrak U`.

## 9. Барьерное подавление

Пусть `B0>0` и `S>0`. Эффективный барьер:

<!-- F11 -->
$$
B_{\mathrm{eff}}=B_0e^{-S},\qquad 0<B_{\mathrm{eff}}<B_0
$$

Это строгое математическое неравенство. Оно не является экспериментальным доказательством LENR.

Отношения, проверяемые машинным ядром:

<!-- F16 -->
$$
\frac{Q_1}{B_1}>1,\qquad \frac{Q_2}{B_2}>1
$$

Тепловое подавление:

<!-- F17 -->
$$
\frac{B_1}{k_BT}\gg1
$$

## 17. Связь с общей теорией в сильной форме

Общая папка `info_graph_theory` задает широкий каркас:

```text
информационный граф
→ лапласиан L
→ спектр
→ структурные функции f_i
→ экспоненциальный регулятор
→ конечный вычислимый сектор
```

Холодный модуль берет из этого каркаса не всю физику, а один прикладной сектор:

```text
информационный граф
→ локальная коллективная среда
→ эффективный барьер
→ барьерное подавление
→ числовой баланс модели
→ машинная проверка внутренней согласованности
```

Поэтому `cold_fusion_graph_theory` не конкурирует с общей папкой.  
Он является ее дочерним сектором.

## 18. Три уровня утверждений

В этом README каждое утверждение должно попадать в один из трех классов.

| Класс | Значение | Пример |
|:--|:--|:--|
| `THEOREM / IDENTITY` | утверждение следует из алгебры или заданных условий | `f2*f5=K` |
| `NUMERIC CHECK` | утверждение подтверждено вычислением | `Q1/B1>1` |
| `MODEL HYPOTHESIS` | физическая трактовка, требующая внешней проверки | LENR как графовое снижение барьера |

Ключевая защита от самообмана состоит в том, что эти классы нельзя смешивать.
Числовой PASS не превращает физическую гипотезу в экспериментальный факт.
Символьное тождество не заменяет эмпирическую верификацию.
Но вместе они создают аккуратный формальный модуль, который можно развивать.

## 20. Почему legacy вынесен из доказательного ядра

Старые прототипы важны как история разработки, но они не должны определять статус теории.
Поэтому они архивированы как текстовые материалы, а в `legacy/` оставлены английские bridge-модули.

Это решает сразу несколько проблем:

- исполняемое ядро становится англоязычным;
- исторические файлы не удаляются;
- старые предположения не смешиваются с текущей проверкой;
- главная проверка идет только через `scripts/verify_all.py`;
- безопасность модуля становится яснее.

Текущий доказательный слой состоит из `scripts/`, `results/`, `reports/` и этого README.

## 21. Машинное ядро подробно

`scripts/cold_fusion_core.py` делает несколько вещей:

1. задает базовые параметры `K`, `p`, `N`;
2. вычисляет структурные функции `f1`-`f6`;
3. проверяет критическое соотношение между `Kp` и `N^(-1/3)`;
4. формирует масштабные величины;
5. проверяет отношения барьера и энергии;
6. вычисляет балансовые показатели;
7. сохраняет JSON;
8. сохраняет markdown-таблицы.

`scripts/cas_verify_readme.py` делает другое:

1. читает `README.md`;
2. ищет display-формулы с ID вида `F01`;
3. сравнивает найденные ID с CAS-регистром;
4. проверяет алгебраические тождества через SymPy;
5. отмечает числовые и определительные формулы;
6. сохраняет отчет `results/cas_report.md`.

`scripts/verify_all.py` соединяет оба слоя:

1. запускает числовое ядро;
2. запускает CAS-проверку;
3. проверяет критические статусы;
4. создает итоговый отчет;
5. печатает `PASS_COLD_FUSION_PACKAGE`, если все прошло.

| `F07` | numeric identity | BIP39: `K+2=8` |
| `F08` | numeric identity | BIP39: `4K(2K-1)=264` |
| `F09` | symbolic identity | критический режим |
| `F10` | limit | абсолютная сходимость модельного ряда |
| `F11` | inequality | барьерное подавление при `B0>0`, `S>0` |
| `F12` | numeric identity | полезная энергия за горизонт |
| `F13` | numeric identity | net-компонент |
| `F14` | definition | главная формула `mathfrak U` |
| `F15` | definition | секторное действие |
| `F16` | numeric inequality | `Q1/B1>1`, `Q2/B2>1` |
| `F17` | numeric inequality | `B1/kBT` велико |
| `F18` | numeric identity | баланс мощности |

Статус `PASS_ALL_TEX_FORMULAS` означает, что найденные в README формулы `F01`-`F18`
совпадают с CAS-регистром и не дают ошибки.

## 27. Почему BIP39-сектор оставлен в cold README

BIP39-сектор не утверждает, что холодный синтез является криптографией.
Он нужен как дискретный тестовый блок общей теории:

- `K=6` должен давать стабильные целые значения;
- эти значения проверяются CAS;
- они связывают частный модуль с общей дискретной структурой;
- они служат контрольным набором, который легко ломается при ошибке параметра.

Это не физическое утверждение о LENR.
Это контрольная математика, встроенная в общий каркас.

---

## 28. Почему абсолютная сходимость важна

Абсолютная сходимость отвечает не за то, что "любой физический процесс реализуется".
Она отвечает за более узкое утверждение: модельный вклад с экспоненциальным регулятором не разлетается при росте индекса.

Внутри общей теории это играет роль стабилизатора:

- полиномиальный рост подавляется экспонентой;
- хвост ряда становится управляемым;
- регуляризованный сектор не дает бесконечного вклада;
- вычисление остается конечным.

Но сходимость сама по себе не доказывает физическую истинность LENR.
Это математическая защита модели от расходимости.

---

## 29. Совместимость с ОТО

Корректная формулировка:

```text
Модуль не противоречит идее, что гравитационный сектор общей теории
должен иметь эффективный ОТО-предел.
```

Некорректная формулировка:

```text
В этом README полностью выведены уравнения Эйнштейна.
```

В холодном модуле ОТО появляется только как сектор совместимости.
Основное содержание cold README не является выводом кривизны пространства-времени.
Он говорит о другом: если общая теория строит гравитационный сектор из спектральной геометрии, то cold-сектор должен быть согласован с этим каркасом и не вводить отдельную физику.

---

## 30. Совместимость со string-like логикой

Корректная формулировка:

```text
Спектральные возбуждения графа можно интерпретировать как модовую картину,
структурно похожую на идею спектра возбуждений.
```

Некорректная формулировка:

```text
В этом README строго доказана теория струн.
```

Важная грань:

- есть спектр;
- есть моды;
- есть возбуждения;
- есть секторное действие;
- но нет worldsheet-формализма;
- нет полного CFT-вывода;
- нет вывода всех струнных состояний.

Поэтому README говорит о совместимости и структурной аналогии, а не о подмене теории струн.

## 31. Четыре взаимодействия

Секторная запись `S_tot = S_em + S_weak + S_strong + S_grav` используется как каркас объединения.
В холодном модуле она важна потому, что LENR-гипотеза не должна быть отдельной пятой физикой.
Она должна быть вложена в существующие сектора:

| Сектор | Роль в общей картине | Роль в cold-модуле |
|:--|:--|:--|
| Электромагнитный | фазовые U(1)-моды | средовой и барьерный фон |
| Слабый | смешивание подпространств | допустимость трансмутационных каналов как гипотезы |
| Сильный | внутренняя стабильность ядерного сектора | барьер и ядерная связность |
| Гравитационный | эффективная геометрия | не доминирует в cold-расчете |

Это не полный лагранжиан Стандартной модели.
Это единая секторная рамка, которую нужно усиливать отдельными доказательными слоями.

## 32. Главная физическая осторожность

Самая опасная ошибка была бы написать:

```text
Машинная проверка README доказывает реальный холодный синтез.
```

Так писать нельзя.

Правильно:

```text
Машинная проверка README доказывает внутреннюю согласованность
формального LENR-сектора при заданных определениях, числах и гипотезах.
```

Разница огромная.
Первое утверждение является экспериментальной претензией.
Второе является математико-вычислительным статусом.

## 34. Что считается разрушением теории

Теорию можно считать разрушенной в одном из случаев:

## 36. Негативные тесты

Хорошая теория должна не только говорить, где эффект возможен, но и где он невозможен.
Для следующего слоя модуля нужны негативные тесты:

| Условие | Ожидаемое поведение |
|:--|:--|
| `S=0` | барьер не подавляется экспоненциально |
| `B0<=0` | формула барьера теряет физический смысл |
| `sigma<=0` | сходимость больше не гарантируется |
| `Q1/B1<=1` | первый канал не проходит энергетический порядок |
| `Q2/B2<=1` | второй канал не проходит энергетический порядок |
| неверный `K` | BIP39-контроль ломается |
| отсутствует `README.md` | CAS-проверка невозможна |
| лишняя `Fxx`-формула без регистрации | CAS-пакет должен падать |

Эти тесты полезны тем, что превращают теорию в проверяемый объект, а не в набор фраз.

## 37. Dimension-checker как следующий слой

Главная техническая задача для усиления физического статуса:

```text
каждая величина должна получить единицу измерения или статус dimensionless.
```

Предлагаемая схема:

| Величина | Тип |
|:--|:--|
| `K` | dimensionless integer |
| `p` | dimensionless graph probability / density parameter |
| `N` | dimensionless cardinal scale |
| `f_i` | dimensionless structural functions |
| `Q1`, `Q2` | energy |
| `B1`, `B2` | energy |
| `kBT` | energy |
| `S` | dimensionless action-like suppression parameter |
| `B_eff/B0` | dimensionless ratio |
| power terms | power |
| energy terms | energy |

После этого можно будет запрещать формулы, где складываются несовместимые размерности.


Правило расширения:

```text
Если добавляется новая display-формула с ID,
```

Правило честности:

```text
Если формула является определением, она должна быть помечена как definition.
```

## 49. Большой итоговый тезис

`cold_fusion_graph_theory` не является доказательством практического холодного синтеза.
Он является большой, структурированной, машинно проверяемой математической моделью LENR-сектора.

Его сильная версия:

```text
В рамках общей информационно-графовой теории задан безопасный cold-сектор,
где барьерное подавление, структурные функции, BIP39-контроль,
абсолютная сходимость и числовой баланс сведены к единому README
и проверяются Python/SymPy-ядром.
```

Его честное ограничение:

```text
Физическая реализация LENR не доказана этим README.
Она остается гипотезой, для которой текущий модуль дает только
формальную, численную и CAS-согласованную рамку.
```

Именно такая формулировка делает теорию не декоративной, а рабочей.
